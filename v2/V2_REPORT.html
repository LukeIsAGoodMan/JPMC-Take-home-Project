Why V2 Was Needed

V1 proved a sound two-layer methodology: CatBoost classifier (PR-AUC 0.70) + 3-segment K-Prototypes. V2 addresses three structural limitations:

Proxy Target
V1 treats income >50K as the only framing
3 Fixed Modes
V1 thresholds are metric-driven, not budget-driven
47% Segment
V1 working-age macro segment is too broad

V2 adds a target hierarchy, budget-aware score bands, exact-lineage Layer 2 sub-segmentation, and optional extension validation — all without changing V1.

Key insightThe biggest improvement opportunity is better targets, not better algorithms. V1's classifier is already strong (PR-AUC 0.70). Moving from Tier 0 to Tier 2 would be more impactful than any model-level tuning.

Full Architecture

Population (199,523) ├── Seniors (40,618, 20%) → retirement ├── Youth (67,462, 34%) → family-indirect └── Working-Age (91,443, 46%) ├── Established Earners (35K) → premium targeting ├── Service Workers (10K) → accessible products └── Younger Workers (46K) → growth / mid-tier

Stability: 3/4 seeds produce 3 sub-segments. Service-worker group is most robust. Boundaries are directional.
